How to Deploy a Static Site to GitHub Pages

Published on June 15, 2023 | By Your Name

GitHub Pages is a free hosting service provided by GitHub that allows you to host static websites directly from your GitHub repository. This makes it an excellent choice for personal websites, project documentation, and landing pages. In this tutorial, we'll walk through the process of deploying a static site to GitHub Pages and connecting it to your custom domain.

Prerequisites

  • A GitHub account
  • Git installed on your computer
  • A static website (HTML, CSS, JavaScript)
  • A custom domain (optional)

Step 1: Create a GitHub Repository

First, you need to create a new repository on GitHub. If you want to create a user or organization site, name your repository username.github.io, where "username" is your GitHub username or organization name.

For a project site, you can use any name for your repository. Project sites will be available at username.github.io/repository-name.

Step 2: Prepare Your Static Site

Make sure your static site is ready for deployment. At a minimum, you should have an index.html file in the root of your project. This will be the homepage of your site.

Your project structure might look something like this:


my-static-site/
├── index.html
├── css/
│   └── styles.css
├── js/
│   └── main.js
└── images/
    └── hero-image.jpg

Step 3: Push Your Code to GitHub

Now, you need to push your code to the GitHub repository you created. Open your terminal and run the following commands:


# Initialize a Git repository (if you haven't already)
git init

# Add all files to the repository
git add .

# Commit the changes
git commit -m "Initial commit"

# Add the remote GitHub repository
git remote add origin https://github.com/username/repository-name.git

# Push the code to GitHub
git push -u origin main

Replace username and repository-name with your GitHub username and repository name.

Step 4: Configure GitHub Pages

Now that your code is on GitHub, you need to enable GitHub Pages:

  1. Go to your repository on GitHub
  2. Click on "Settings"
  3. Scroll down to the "GitHub Pages" section
  4. Under "Source", select the branch you want to deploy (usually "main" or "master")
  5. Click "Save"

GitHub will provide you with a URL where your site is published (e.g., https://username.github.io/repository-name).

Step 5: Connect Your Custom Domain (Optional)

If you have a custom domain, you can connect it to your GitHub Pages site:

  1. In your repository's "Settings", scroll to the "GitHub Pages" section
  2. Under "Custom domain", enter your domain name (e.g., www.example.com)
  3. Click "Save"

Next, you need to configure your domain's DNS settings. Go to your domain registrar's website and add the following records:

For an apex domain (example.com):


A     @     185.199.108.153
A     @     185.199.109.153
A     @     185.199.110.153
A     @     185.199.111.153

For a www subdomain:


CNAME     www     username.github.io

It may take up to 24 hours for DNS changes to propagate.

Step 6: Enable HTTPS (Recommended)

For security reasons, it's recommended to enable HTTPS for your site:

  1. In your repository's "Settings", scroll to the "GitHub Pages" section
  2. Check the "Enforce HTTPS" option

Note that you may need to wait a few hours after setting up your custom domain before you can enable HTTPS.

Conclusion

Congratulations! You've successfully deployed your static site to GitHub Pages. Your site is now live and accessible to the world. Any changes you push to your repository will be automatically deployed to your site.

GitHub Pages is a powerful and free way to host static websites. It's perfect for personal portfolios, project documentation, and simple landing pages. Combined with a static site generator like Jekyll, Hugo, or 11ty, you can create sophisticated websites without the need for a traditional content management system.
